# 服务端部署

如果你在 github 搜索过 arcaea，你一定发现了最前面的 arcaea-server 其由 Lost-MSth 提供，因为本身属于一个开源项目，部署教程已经相对完整，细见：

Lost-MSth/Arcaea-server: 一个微型的 Arcaea 本地服务器 A small local server for Arcaea (github.com)

官方使用说明

这部分教程比较基础，对于一般学习了 python 的人，基本可以独立完成，故不多阐述

# 代理的安装

这一步的目的其实就是引导客户端连接到自己的服务器，笼统来讲，就是 dns 劫持，将原本发送到官方服务器的流量发到自己搭建的服务器，让客户端和自我搭建的服务器通讯，这一步的实现方法很多，例如自建 dns 服务器，代理的 map remote 实现域名重定向，或者直接修改客户端 lib 的地址，这里我们用 map remote 实现。

csdn 相关教程

这里笔者个人搭建的过程中发现，ssl proxying settings 不能打开，否则服务端会出现 400 错误，原因未知。

官方 api 地址：arcapi-v2.lowiro.com

# ssl pinning 的拆除

这一步也是最难的一步，因为他涉及到逆向，而且国内几乎没有任何资料，如前文所言，我们要引导客户端连接到服务端，但 arc 在 3.6.2 加入了证书进行验证，如果没有破除 arc 证书的验证，客户端就无法连接服务端。

ssl pinning 的相关细节

笔者之前在国内找到了一篇文章来自博主大 A 的

Arcaea 的逆向分析 | 熊猫人 - 大 A (ashenone66.cn)

但本人实践之后，博文所说的 openssl 函数在最新版本已经混淆，无法再次使用，而且 frida 在非 root 机型中的使用也及其困难。

笔者在这里卡住了近两个月时间，期间也尝试过向 lost 以及大 a 进行求助，但因为一些原因，最终没有解决。

幸运的是，笔者在一次偶然间，通过纯英文搜索 sslpinning arcaea 的关键词之后，找到了一些资料，github 上有一篇 wiki，解决了我的烦恼。

wiki 地址

通过这篇文章，在客户端的修改便不成问题。
